Prevent cheating

On Hyperskill, sometimes we need to check that the student's code doesn't use a particular method. For instance, when the task is to implement the computation of factorial manually instead of using the function factorial() from the math module. How could we do that?

In fact, it is really simple. We can override math.factorial() to do something else: raise an error or print a message to the student. To do so, we firstly define a function that does what we want (e.g. prints a message) and then assign this function to math.factorial. Now, if the student tries to use math.factorial, our custom function will be raised. This happens because when there exist multiple functions with the same name, the later one always overrides the prior.

For this task, imagine that you are a Hyperskill content creator and implement the example with the factorial yourself. Your program should override math.factorial() and make it print the message "Don't cheat!" instead of actually calculating a factorial.
